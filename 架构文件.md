# PaperMind 架构文档

## 1. 项目概述

PaperMind 是一个论文解析系统，采用 **Spring Cloud 微服务架构**，支持论文上传、AI分析和团队协作功能。

## 3. 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|

| **服务通信** | OpenFeign + LoadBalancer | 远程调用 |

| **前端** | React | 用户界面 |

## 5. 服务详解

### 5.1 User Service（用户服务）

**端口**: 8081 | **职责**: 用户管理

```
User.java (实体类)
    ├── id: Long
    ├── username: String
    ├── email: String
    └── role: String

UserRepository.java (数据访问层)
    └── 继承 JpaRepository<User, Long>，提供 CRUD 方法

UserService.java (业务逻辑层)
    └── 处理用户相关业务逻辑

UserController.java (REST控制器)
    └── 提供 HTTP API: GET /users/{id}
```

### 5.2 Paper Service（论文服务）

**端口**: 8082 | **职责**: 论文管理，调用 AI 引擎

```
Paper.java (实体类)
    ├── id: Long
    ├── title: String
    ├── author: String
    ├── content: String
    └── analysisResult: String

PaperService.java
    └── 关键方法: analyzePaper(paperId) → 调用 AiEngineFeignClient

AiEngineFeignClient.java (Feign客户端)
    └── @FeignClient(name = "ai-engine")
    └── analyzePaper(paperId) → 调用 AI 服务的 /ai/analyze/{paperId}
```

### 5.3 AI Engine（AI引擎）

**端口**: 8083 | **职责**: 论文 AI 分析

```
AiService.java
    └── analyzePaper(paperId) → 执行论文分析逻辑

AiController.java
    └── POST /ai/analyze/{paperId}
```

### 5.4 Gateway Service（网关服务）

**端口**: 8080 | **职责**: 统一入口，路由转发

```
GatewayConfig.java
    └── 路由规则:
        /users/**  → lb://user-service
        /papers/** → lb://paper-service
        /ai/**     → lb://ai-engine
```

## 6. 核心概念解释

### 6.1 服务注册与发现（Nacos）

```
启动流程:
1. 服务启动时向 Nacos 注册自己的地址和端口
2. Nacos 记录: user-service → 192.168.1.100:8081
3. 其他服务可以从 Nacos 发现 user-service 的地址
```

### 6.2 API Gateway（网关）

```
客户端请求流程:
1. 客户端请求: GET http://localhost:8080/papers/1
2. 网关根据路由规则将请求转发到 paper-service
3. paper-service 返回响应给网关
4. 网关将响应返回给客户端

优势: 客户端只需知道网关地址，无需知道各服务地址
```

### 6.3 OpenFeign（服务间通信）

```
paper-service 调用 ai-engine 的方式:
1. 定义 Feign 接口 (AiEngineFeignClient)
2. 像调用本地方法一样调用远程服务
3. LoadBalancer 负责负载均衡

示例:
@FeignClient(name = "ai-engine")
public interface AiEngineFeignClient {
    @PostMapping("/ai/analyze/{paperId}")
    Map<String, Object> analyzePaper(@PathVariable Long paperId);
}

// 在 PaperService 中使用
@Autowired
private AiEngineFeignClient aiEngineFeignClient;

public void analyze(Long paperId) {
    Map<String, Object> result = aiEngineFeignClient.analyzePaper(paperId);
}
```

## 7. Spring Cloud 微服务核心注解

| 注解 | 位置 | 作用 |
|------|------|------|
| `@SpringBootApplication` | 启动类 | 标识为 Spring Boot 应用 |
| `@RestController` | Controller | REST API 控制器 |
| `@Service` | Service | 业务逻辑组件 |
| `@Repository` | Repository | 数据访问组件 |
| `@Entity` | 实体类 | 映射数据库表 |
| `@FeignClient` | Feign接口 | 声明式 HTTP 客户端 |
| `@EnableFeignClients` | 启动类 | 启用 Feign 客户端 |

## 9. 学习路径建议

### 第一阶段：Spring Boot 基础（1-2天）

从 **user-service** 开始学习标准分层架构：

1. `UserApplication.java` - 启动类，理解 `@SpringBootApplication`
2. `User.java` - 实体类，理解 `@Entity`, JPA 映射
3. `UserRepository.java` - 数据访问，理解 Spring Data JPA
4. `UserService.java` - 业务层，理解 `@Service` 和依赖注入
5. `UserController.java` - 控制器，理解 `@RestController`

### 第二阶段：API Gateway（半天）

学习 **gateway-service**：
- `GatewayConfig.java` - 路由配置
- 理解 `lb://` 负载均衡前缀

### 第三阶段：服务间通信（半天）

学习 **paper-service** 中的 Feign：
- `AiEngineFeignClient.java` - 声明式 HTTP 客户端
- 理解 `lb://` 如何实现负载均衡调用

### 第四阶段：理解完整业务流程（半天）

1. 用户上传论文 → PaperController
2. PaperService 调用 AiEngineFeignClient
3. AI Engine 处理分析请求

## 10. 快速启动

```bash
# 1. 启动 Nacos（需要先下载）
cd nacos/bin
startup.cmd -m standalone  # Windows

# 2. 启动后端服务
cd backend
mvn spring-boot:run -pl user-service      # 8081
mvn spring-boot:run -pl paper-service     # 8082
mvn spring-boot:run -pl ai-engine         # 8083
mvn spring-boot:run -pl gateway-service   # 8080

# 3. 启动前端
cd frontend && npm start  # 3000

# 4. 访问
http://localhost:3000
```
