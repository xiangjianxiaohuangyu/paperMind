│   └── User.java                # JPA 实体类
├── dto/
│   ├── LoginRequest.java        # 登录请求 DTO
│   ├── RegisterRequest.java      # 注册请求 DTO
│   └── AuthResponse.java        # 认证响应 DTO
├── config/
│   ├── SecurityConfig.java      # Spring Security 配置
│   └── CustomUserDetailsService.java
├── utils/
│   └── JwtUtils.java            # JWT 令牌工具类

### 技术栈
| 技术 | 说明 |
|------|------|
| **Spring Boot** | Web 框架 |
| **Spring Security** | 安全认证框架 |
| **Spring Data JPA** | ORM 持久层框架 |
| **JWT (jjwt)** | JSON Web Token 无状态认证 |
| **BCrypt** | 密码加密算法 |
| **MySQL** | 数据库 |
| **Maven** | 项目构建工具 |

### 核心知识点学习

#### 1. 分层架构 (MVC/Spring Boot 标准分层)
```
Controller → Service → Repository → Database
   ↑           ↑          ↑
 接收请求    处理业务     操作数据
```

#### 2. JPA 实体注解
```java
@Entity                          // 标识为 JPA 实体
@Table(name = "users")           // 对应数据库表名
@Id                             // 主键
@GeneratedValue                  // 主键生成策略
@Column                          // 列配置（可空、唯一等）
```

#### 3. Spring Security 配置
```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    // BCryptPasswordEncoder: 密码加密
    // CORS: 跨域资源共享
    // SessionCreationPolicy.STATELESS: 无状态会话（JWT 使用）
}
```

#### 4. JWT 认证流程
```
用户登录 → 验证密码 → 生成 Token → 返回给前端
前端请求 →携带 Token → 验证 Token → 返回数据
```

#### 5. RESTful API 设计
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /users/register | 用户注册 |
| POST | /users/login | 用户登录 |
| GET | /users/{id} | 获取用户信息 |

### 相关注释学习
代码中包含了详细的中文注释，解释了每层的职责：
- **Controller**: "前后端的大门 / 入口"，只做"接收请求、调用Service、返回结果"
- **Service**: "项目的逻辑中枢"，"处理所有业务逻辑"
- **Entity**: "数据搬运工"，"把数据库的一行数据，变成 Java 对象"
